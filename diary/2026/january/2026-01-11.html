STATE OF CURSOR TODAY AND THE UNITED STATES OF AMERICA

I was vibe coding along with Cursor today.Agent mode.
It was generating a markdown file for each prompt.
In one prompt, I asked it to "give pros and cons of each, lets discuss".
Even then, it went ahead and created a markdown file.
And the same think was in the response of each prompt.
Thats the same content twice, in the output.
Feel like these apps are hardcoded to generate as much text as possible.
Not to mention, each query takes in the whole history again. Each query takes in the whole cumulative contxt .
So lets assume , in theory, there is a chat between two people Q and A.
Lets assume the conversation is liek : Q1,A1,Q2,A2,Q3,A3,Q4,A4.

But with LLMs, the conversation is like Q1,A1,(Q2+Q1),A2,(Q1+Q2+Q3),A3,(Q1+Q2+Q3+Q4),A4.

Bet these companies are justifying their capex spends based on metrics like aggreagte input token count and output otken count.
I know there is a reponses api which can take in previous_response_id but this is not the default in cursor.ITs still on chat completions API.
